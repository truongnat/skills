# Simple Skills

A minimal set of skills and rules for AI agents, focused on clear workflows,
readability, and extensibility. The kit stays contract-first and install stays
small by default (`core` profile) — opt into office/frontend/backend packs or
`all` when needed.

## Goals

- Help agents work through a clear lifecycle: think → design → plan → execute → review → done.
- Encourage artifact and session creation instead of running tasks vaguely.
- Easy to install into any repo by creating a `.agents` directory.
- Keep host installs lean: default `core`, shared preamble, thin `AGENTS.md`,
  executable artifact schema checks before review/done.

## Installation

Run from the project directory where you want to install the skills. The script
places `AGENTS.md` at the project root and supporting files under `.agents/`:

```text
AGENTS.md                 # Short entrypoint (points at AGENT_POLICY.md)
.agents/
├── settings.yaml
├── PRJ_REFERENCE.md      # Generated by the init skill
├── SKILL_PREAMBLE.md     # Shared Language + Memory
├── AGENT_POLICY.md       # Full mandatory policy
├── DESIGN_SYSTEM.md
├── CODE_COMMENTS.md
├── THIRD_PARTY_SKILLS.md
├── tools/                # session, decision-server, validate_artifacts, …
└── skills/
    ├── brainstorming/
    │   ├── SKILL.md
    │   ├── agents/
    │   │   └── openai.yaml
    │   ├── steps/         # Ordered, fail-closed workflow
    │   └── templates/     # Mandatory artifact seeds
    ├── planning/
    └── ...
```

The installer only copies skills and does not install Python packages.
Python-based skills create a private `.venv` from their own `requirements.txt`
on first use.

Choose an install profile to avoid copying unused third-party packs:

| Profile | Contents |
| --- | --- |
| `core` | First-party lifecycle/workflow only (**default**) |
| `office` | `core` + office Python skills |
| `frontend` | `core` + web/design/expo third-party skills |
| `backend` | `core` + backend/API/DB/security/devops third-party skills |
| `all` | Every skill (opt-in full pack) |

Comma-separated profiles are unioned (for example `office,frontend`). Set
`SIMPLE_SKILLS_PROFILE` or pass `--profile` / `-Profile`. Use `--profile all`
only when you need the full third-party pack.

`.agents/settings.yaml` configures agent behavior. Set `language: en` or
`language: vi` to control the language of agent replies and saved artifacts.
Set `rules.reports.output_format` to `markdown` (default) or `html` to choose
the representation of lifecycle reports while keeping the same contract
sections and logical filenames. HTML uses one enterprise theme (Tailwind CDN + anime.js CDN for static
open; decision-server for interactive choices). See `.agents/DESIGN_SYSTEM.md`.
Reinstalling preserves an existing `settings.yaml`.

After installation, run the `init` skill once. It inspects the project and
generates `.agents/PRJ_REFERENCE.md`, then merges confirmed branch, commit, PR,
report, and code-comment conventions into `settings.yaml`. Re-run `init` to
refresh; use force mode only when a full regeneration is intended.

If root `AGENTS.md` already exists, choose one conflict policy:

- `prompt` (default): ask before replacement; keep the file when no interactive
  terminal is available.
- `replace`: replace it without prompting.
- `skip`: always keep it.

Set `SIMPLE_SKILLS_AGENTS_MODE=prompt|replace|skip`, or pass the corresponding
installer option.

### Linux / macOS

```bash
# Default: core profile
curl -fsSL https://raw.githubusercontent.com/truongnat/simple-skills/main/install.sh | bash

# Non-interactive replacement
curl -fsSL https://raw.githubusercontent.com/truongnat/simple-skills/main/install.sh \
  | bash -s -- --agents-mode replace

# Lifecycle + office only
curl -fsSL https://raw.githubusercontent.com/truongnat/simple-skills/main/install.sh \
  | bash -s -- --profile office --agents-mode replace

# Full skill tree (previous default behavior)
curl -fsSL https://raw.githubusercontent.com/truongnat/simple-skills/main/install.sh \
  | bash -s -- --profile all --agents-mode replace
```

### Windows (PowerShell)

```powershell
# Default: core profile
irm https://raw.githubusercontent.com/truongnat/simple-skills/main/install.ps1 | iex

# Non-interactive replacement
$env:SIMPLE_SKILLS_AGENTS_MODE = "replace"
irm https://raw.githubusercontent.com/truongnat/simple-skills/main/install.ps1 | iex

# Office pack
$env:SIMPLE_SKILLS_PROFILE = "office"
$env:SIMPLE_SKILLS_AGENTS_MODE = "replace"
irm https://raw.githubusercontent.com/truongnat/simple-skills/main/install.ps1 | iex
```

### Install from cloned repo

If you already cloned this repo, run the install script directly:

```bash
# Linux / macOS (core by default)
./install.sh --agents-mode prompt

# Office + frontend packs only
./install.sh --profile office,frontend --agents-mode replace

# Windows (PowerShell)
./install.ps1 -AgentsMode prompt

# Windows (PowerShell) with profile
./install.ps1 -Profile office -AgentsMode replace

# Windows (CMD)
install.cmd -AgentsMode prompt
```

## Available Skills

First-party workflow skills (installed with `core`):

| Skill | Purpose |
| --- | --- |
| `scaffold` | Greenfield project bootstrap (stack, structure, tooling) then hand off to `init` |
| `init` | Collect project context and generate `.agents/PRJ_REFERENCE.md` plus project settings |
| `brainstorming` | Step workflow: seed DISCUSSION → frame + Spec quality → options → recommend → self-check |
| `business-analysis` | Step workflow: seed BUSINESS_ANALYSIS → frame + Spec quality → stories/rules/AC → self-check |
| `basic-design` | System-level design: boundaries, components, flows, interfaces, data ownership |
| `detail-design` | Implementable design: contracts, data model, sequences, rules/operations (dynamic depth) |
| `planning` | Step workflow: seed PLAN/TASKS → decision + Spec quality gate → strategy → micro-tasks → self-check |
| `sync` | Sync codebase understanding, git state, and context (read-only by default) |
| `execution` | Record execution steps and changes made |
| `review` | Review correctness, regression risk, security, and maintainability |
| `done` | Summarize work, generate PR message, and handoff |
| `investigate` | Debug, analyze, or trace root cause before deciding to implement |
| `research` | Research internal or external sources before making decisions |
| `review-pr` | Review pull requests or diffs in a structured way |
| `tester` | Create test cases, verify, and record evidence |
| `docs` | Enterprise wiki set (SRS/architecture/ADRs/HLD/LLD/…) under `.agents/wiki/` |

### First-party office skills (Python)

Install with `--profile office` (or `all`):

| Skill | Purpose |
| --- | --- |
| `xlsx` | Inspect/create/edit/validate `.xlsx` with supported-lossless coverage |
| `docx` | Inspect/create/edit/validate `.docx` with supported-lossless coverage |
| `pptx` | Inspect/create/edit/validate `.pptx` with supported-lossless coverage |
| `pdf` | Inspect/create/edit/validate PDF with supported-lossless coverage |

These skills use Python CLIs under `skills/<name>/scripts/cli.py` plus shared
helpers in `skills/office-common/`. Create/edit requires `coverage_ratio == 1.0`
before publish. Run `python .agents/skills/<name>/scripts/setup.py` on first
use; dependencies stay in that skill's `.venv`. Repository development can use
`pip install -e ".[dev]"`.

## Quick Start

After installation, agents read root `AGENTS.md`, then `.agents/AGENT_POLICY.md`
and `.agents/SKILL_PREAMBLE.md`. Each task starts by reading
`.agents/settings.yaml` and the init-generated `.agents/PRJ_REFERENCE.md`, then
creates its own session:

```text
.agents/sessions/<Task-<number>-<short-description>>/
├── OVERVIEW.md        # developer landing page (status + charts)
├── DISCUSSION.md      # brainstorming
├── BUSINESS_ANALYSIS.md # business-analysis
├── BASIC_DESIGN.md    # basic-design
├── DETAIL_DESIGN.md   # detail-design
├── PLAN.md            # planning (strategy)
├── TASKS.md           # planning (detailed tasks)
├── EXECUTION.md       # execution
├── REVIEW.md          # review
├── TESTCASES.md       # tester
└── DONE.md            # done
```

Resolve the active session and real progress with:

```bash
bash .agents/tools/session/session.sh current
bash .agents/tools/session/session.sh status
```

Before `review` Ready / `done` complete, validate artifact headings:

```bash
python .agents/tools/session/validate_artifacts.py
```

Reports are skim-first: executive summary, developer overview panel, and
Mermaid charts when they improve scanability. Detail stays after the overview.

Brainstorming, business-analysis, and planning are fail-closed step workflows.
Their artifact Step ledger must be updated in order, and agents must complete
the Feasibility, Correctness, and Capability recommendations review before
writing downstream scope, stories/acceptance criteria, recommendations, or
task cards. Blocking findings stop the workflow for focused user questions.

Visual decision HTML pages support dual load: static open via CDN tags, or
local serve for interactive choices:

```bash
python .agents/tools/session-serve/serve.py
```

The wrapper finds the latest session containing `VISUAL_DECISION.html`, opens
it, and writes logs under that session. Static preview still works because the
HTML itself includes Tailwind + anime.js CDN tags. Read the latest user choice with:

```bash
python .agents/tools/choice-reader/read.py --issue-id <issue-id>
```

Screen recordings can be converted into image evidence for
business-analysis, investigation, or testing:

```bash
python .agents/tools/video-keyframes/extract.py evidence/bug.webm \
  --output-dir .agents/sessions/<task>/video-evidence
```

This tool requires `ffmpeg` and `ffprobe` on `PATH`.

Each first-party workflow skill's **binding Contract** lives in `SKILL.md`
under `## Contract (mandatory)`. Third-party skills retain their upstream
instructions. Agents must always follow `SKILL.md`, not rely on metadata alone.

For the short entrypoint see [docs/AGENTS.md](docs/AGENTS.md); for full policy
see [docs/AGENT_POLICY.md](docs/AGENT_POLICY.md).

## Development

Installing skills requires only `curl` on Linux/macOS or PowerShell on Windows.

Repo checks (also run in GitHub Actions CI):

```bash
pip install -e ".[dev]"
python scripts/validate_skills.py
python scripts/validate_office_skills.py
python scripts/resolve_install_profile.py --source . --check
pytest -q
```

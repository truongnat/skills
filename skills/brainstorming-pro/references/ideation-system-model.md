# Ideation System Model

## Overview

The brainstorming process follows a structured flow from raw ideas to converged, actionable options. This model ensures systematic exploration and disciplined convergence.

## Three-Phase Flow

### Phase 1: Divergence (Raw Idea Generation)

**Goal:** Generate maximum breadth of ideas without judgment.

- **Input:** Problem/opportunity statement, constraints
- **Process:** 
  - Suspend criticism
  - Use Socratic questioning to expand thinking
  - Explore constraints as variables, not blockers
  - Generate quantity over quality initially
- **Output:** Raw idea list, assumption inventory

### Phase 2: Structured Exploration

**Goal:** Refine and deepen understanding of promising ideas.

- **Input:** Raw idea list, constraint inventory
- **Process:**
  - Surface hidden assumptions explicitly
  - Test assumption validity
  - Explore constraint variations
  - Assess technical feasibility (with domain skills)
  - Identify interdependencies between ideas
- **Output:** Structured idea matrix with assumptions, feasibility notes, risk flags

### Phase 3: Convergence (Filtering to Actionable Options)

**Goal:** Select and prioritize options for implementation.

- **Input:** Structured idea matrix
- **Process:**
  - Apply verifiable convergence criteria
  - Filter against constraints
  - Rank by impact/feasibility
  - Document rationale for inclusions/exclusions
- **Output:** Prioritized option set with implementation readiness assessment

## Feedback Loops

### Divergence → Exploration Loop

When exploration reveals gaps in idea space:
- Return to divergence with refined questions
- Focus on identified blind spots
- Avoid rehashing exhausted areas

### Exploration → Convergence Loop

When convergence criteria reveal insufficient depth:
- Return to exploration for missing data
- Target specific assumption testing
- Validate feasibility with domain skills

## Quality Gates

### Phase 1 Gate

- Minimum 3 distinct approaches generated
- Assumption inventory started
- No premature filtering occurred

### Phase 2 Gate

- All major assumptions surfaced
- Feasibility assessed for top options
- Constraint implications explored

### Phase 3 Gate

- Convergence criteria applied consistently
- Rationale documented for decisions
- Options map to implementation readiness

## Failure Modes

- **Premature convergence:** Jumping to solution before exploration
- **Solution-first thinking:** Starting with answer, then justifying
- **Assumption blindness:** Not making constraints explicit
- **Groupthink:** Converging without structured divergence

## Integration Points

- **`business-analysis-pro`:** Requirements validation before ideation
- **`planning-pro`:** Implementation planning after convergence
- **Domain `*-pro` skills:** Technical feasibility assessment in Phase 2
- **`strategic-consulting-pro`:** Portfolio-level option evaluation

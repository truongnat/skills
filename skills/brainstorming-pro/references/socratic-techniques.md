# Socratic Questioning Techniques

## Overview

Socratic questioning uses disciplined inquiry to refine ideas, surface assumptions, and deepen understanding. Unlike assertion-based brainstorming, it uses questions to guide exploration.

## Core Question Types

### 1. Clarification Questions

**Purpose:** Ensure shared understanding of the problem.

- "What do you mean by [term]?"
- "Can you give an example of that?"
- "How would you define success in this context?"
- "What's the core problem we're solving?"

### 2. Assumption-Surfacing Questions

**Purpose:** Make hidden constraints explicit.

- "What are we assuming about [X]?"
- "What would have to be true for this to work?"
- "What constraints are we treating as fixed that could be variables?"
- "What if that assumption were false?"

### 3. Evidence-Testing Questions

**Purpose:** Validate or challenge beliefs.

- "What evidence supports this approach?"
- "What would disprove this idea?"
- "How would we test this assumption?"
- "What data do we need to validate this?"

### 4. Alternative-Generation Questions

**Purpose:** Force exploration beyond the obvious.

- "What's another way to think about this?"
- "What if we removed constraint [X]?"
- "How would [different stakeholder] approach this?"
- "What's the opposite of this idea?"

### 5. Implication-Exploration Questions

**Purpose:** Understand downstream consequences.

- "What happens if we do this?"
- "What are the second-order effects?"
- "Who/what would be affected by this change?"
- "What breaks if we implement this?"

### 6. Perspective-Shifting Questions

**Purpose:** Break out of current mental models.

- "How would [competitor/user/regulator] view this?"
- "What does this look like from [different timeframe]?"
- "What if we had zero budget/infinite time?"
- "What's the simplest version of this?"

## Question Sequences

### Sequence 1: Problem Clarification

1. "What's the specific problem we're solving?"
2. "For whom is this a problem?"
3. "What does 'better' look like?"
4. "What are we not trying to solve?"

### Sequence 2: Assumption Testing

1. "What are we assuming is true about [X]?"
2. "What evidence supports that assumption?"
3. "What would change if that assumption were false?"
4. "How can we test this assumption quickly?"

### Sequence 3: Alternative Exploration

1. "What's the obvious solution?"
2. "What if we couldn't use that solution?"
3. "What's a completely different approach?"
4. "What constraints could we remove to enable new options?"

### Sequence 4: Convergence Preparation

1. "Which options are actually feasible?"
2. "What criteria should we use to evaluate these?"
3. "What are the risks of each option?"
4. "What information are we still missing?"

## Anti-Patterns

### Leading Questions

- **Bad:** "Don't you think we should use [solution]?"
- **Good:** "What are the trade-offs of [solution] vs alternatives?"

### Rhetorical Questions

- **Bad:** "Isn't it obvious that [X] won't work?"
- **Good:** "What would need to be true for [X] to work?"

### Judgment-Laden Questions

- **Bad:** "Why would anyone choose that approach?"
- **Good:** "What scenarios would make that approach optimal?"

## Facilitation Tips

### Use in Groups

- Pause after each question to allow thinking time
- Rotate who asks questions to avoid dominance
- Write questions on board to track exploration path
- Return to core questions when conversation drifts

### Use in Async

- Present question sequences as prompts
- Allow time between questions for reflection
- Document responses to build shared understanding
- Use question threading to deepen specific areas

### Use with Domain Skills

- Let domain `*-pro` skills answer technical feasibility questions
- Use Socratic questions to frame what domain skills should evaluate
- Document domain skill responses as evidence
- Return to ideation when domain skills reveal new constraints

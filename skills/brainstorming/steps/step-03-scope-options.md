# Step 03 — Scope and options

## Goal

Lock scope and fill the options comparison matrix in `DISCUSSION.md`.

## Rules

- Edit **only** `DISCUSSION.md` (scope + options sections).
- Before editing, re-check Issue triage and Clarification checkpoint. If any
  Critical/blocking item is unresolved, **stop and ask**; do not scope around it.
- Scope in / scope out / non-goals must not contradict each other.
- Options table: at least **one** option; prefer **2+** when trade-offs exist.
- Columns: Option, Pros, Cons, Effort, Risk, Reversible?, How to verify.
- Do **not** write the final Recommendation/Handoff yet (step-04) — draft row content only.
- Lite Mode: 1–2 options still OK; still use the template table.
- If visual triage says `html-recommended` and the user confirmed it, seed
  `VISUAL_DECISION.html` from the template before comparing options. If not
  confirmed, stop and ask. Use a diagram/table instead when it is sufficient.
- Serve the page with `.agents/tools/session-serve/serve.py <session>`, wait for
  the user choice, read it with the choice-reader for that session and Issue
  ID, then record the answer in Clarification checkpoint.

## Fill these sections

1. Scope in
2. Scope out
3. Non-goals
4. Options considered (matrix)

## Done when

- [ ] Scope in and scope out are both non-empty (or explicitly N/A with reason).
- [ ] Options matrix has ≥1 real option (not only `A — _(name)_` placeholders).
- [ ] Each option has a verify method.
- [ ] Every issue driving an option is resolved or explicitly non-blocking.
- [ ] Visual format matches the triage; HTML was not generated by default.

# Business analysis — anti-patterns

## Untestable requirements

- “World-class”, “intuitive” without measure.
- **Fix:** SMART criteria, acceptance tests, baseline metrics.

## Single workshop as “signed off”

- No written traceability.
- **Fix:** Decision log, versioned doc, named approvers.

## Solution before problem

- Jumping to features without outcome.
- **Fix:** Problem → metric → option → recommendation.

## Ignoring non-functional requirements

- Only happy-path user stories.
- **Fix:** Explicit NFRs (performance, availability, privacy).

## AI-generated requirements without review

- Hallucinated regulations or standards.
- **Fix:** Human validation; cite authoritative sources.

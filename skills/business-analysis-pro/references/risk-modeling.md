# Risk modeling

Structured risks beat narrative worry lists.

## Fields (minimum)

| Field | Purpose |
|-------|---------|
| **ID** | Stable reference (R-01) |
| **Description** | Cause + effect |
| **Category** | Schedule, tech, compliance, stakeholder, operational, … |
| **Probability** | Low / med / high — or **qualitative** with rationale |
| **Impact** | On scope, schedule, reputation, legal — severity |
| **Mitigation** | Preventive or detective actions |
| **Owner** | Named role |
| **Due / review** | When mitigation must be in place or risk revisited |

## Matrix

Plot **probability × impact** for triage; **avoid** debating numeric precision when data is thin — use **explicit assumptions** instead.

## Issues vs risks

- **Issue** — Already happening.
- **Risk** — May happen; mitigations reduce probability or impact.

# Adaptive Replanning

## Overview

Adaptive replanning adjusts the execution plan when circumstances change, blockers occur, or new information emerges. This ensures the plan remains relevant and achievable.

## When to Replan

### Replan Triggers

**Blockers:**
- Critical blocker that cannot be resolved quickly
- Multiple high-severity blockers
- Blocker with no clear resolution path

**New Information:**
- Requirements change significantly
- New constraints discovered
- Technical approach deemed infeasible
- External dependencies change

**Performance Issues:**
- Consistently missing estimates
- Quality issues requiring rework
- Resource constraints tighter than expected
- Timeline no longer achievable

**Stakeholder Input:**
- Stakeholder requests changes
- Priority shifts
- Scope changes
- Timeline adjustments

### Replan Thresholds

**Immediate Replan:**
- Critical blocker
- Major requirement change
- Timeline no longer achievable

**Planned Replan:**
- Regular review cycle (e.g., weekly)
- Milestone completion
- Significant progress variance

**Deferred Replan:**
- Minor variance within acceptable range
- Low-severity blocker
- Small requirement clarification

## Replanning Process

### Step 1: Assess Situation

**Actions:**
- Identify what changed
- Assess impact on plan
- Determine urgency
- Gather relevant data

**Questions:**
- What triggered the need to replan?
- What is the impact on timeline?
- What is the impact on scope?
- What is the impact on resources?

**Output:** Situation assessment

### Step 2: Identify Options

**Actions:**
- Brainstorm possible responses
- Evaluate each option
- Consider trade-offs
- Identify constraints

**Options:**
- Adjust timeline
- Adjust scope
- Adjust resources
- Change approach
- Cancel or defer work

**Output:** Option analysis

### Step 3: Select Approach

**Actions:**
- Select best option based on constraints
- Consider stakeholder priorities
- Validate feasibility
- Get approval if needed

**Criteria:**
- Minimizes impact on goals
- Maximizes value delivered
- Maintains feasibility
- Aligns with stakeholder priorities

**Output:** Selected approach

### Step 4: Update Plan

**Actions:**
- Modify task list as needed
- Update dependencies
- Adjust estimates
- Update milestones
- Document changes

**Output:** Updated plan

### Step 5: Communicate Changes

**Actions:**
- Inform team of changes
- Inform stakeholders of changes
- Explain rationale
- Get confirmation

**Output:** Change communication

### Step 6: Resume Execution

**Actions:**
- Resume execution with updated plan
- Track against new baseline
- Monitor for further changes
- Document lessons learned

**Output:** Resumed execution

## Replanning Strategies

### Scope Reduction

**Approach:** Remove non-essential tasks or features

**When to Use:**
- Timeline pressure
- Resource constraints
- Lower priority items identified

**Benefits:**
- Maintains timeline
- Focuses on high-value items
- Reduces complexity

**Drawbacks:**
- Delivers less value
- May disappoint stakeholders
- May need to add back later

### Timeline Extension

**Approach:** Extend project timeline

**When to Use:**
- Critical path cannot be compressed
- Scope cannot be reduced
- Stakeholders accept delay

**Benefits:**
- Maintains full scope
- Reduces pressure
- Allows for quality

**Drawbacks:**
- May impact business goals
- May increase cost
- Stakeholder resistance

### Resource Addition

**Approach:** Add more resources to project

**When to Use:**
- Resources available
- Tasks can be parallelized
- Timeline fixed

**Benefits:**
- Maintains scope and timeline
- Reduces individual workload

**Drawbacks:**
- May increase cost
- Coordination overhead
- Ramp-up time

### Approach Change

**Approach:** Change technical or process approach

**When to Use:**
- Current approach not working
- Better approach identified
- Technical constraints discovered

**Benefits:**
- May solve root cause
- May improve efficiency

**Drawbacks:**
- May require rework
- May introduce new risks
- May delay progress

## Common Mistakes

### Replanning Too Frequently

**Pattern:** Replanning for every minor variance

**Problem:** Plan churn, loss of focus, team confusion

**Fix:** Set clear replan thresholds, avoid knee-jerk reactions

### Not Replanning When Needed

**Pattern:** Sticking to plan despite clear need for change

**Problem:** Plan becomes obsolete, execution fails

**Fix:** Monitor for replan triggers, act when thresholds met

### Replanning Without Analysis

**Pattern:** Making changes without understanding impact

**Problem:** Changes may make things worse

**Fix:** Always assess situation before replanning

### Not Communicating Changes

**Pattern:** Updating plan without informing stakeholders

**Problem:** Misalignment, surprises, stakeholder dissatisfaction

**Fix:** Always communicate changes with rationale

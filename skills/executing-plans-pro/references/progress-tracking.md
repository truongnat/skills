# Progress Tracking

## Overview

Progress tracking monitors execution against the plan, identifying variances, blockers, and trends. This enables early intervention and plan adjustment.

## Tracking Metrics

### Completion Metrics

**Tasks Completed:** Number of tasks completed
**Tasks Remaining:** Number of tasks remaining
**Completion Percentage:** (Completed / Total) * 100
**On-Track Rate:** Percentage of tasks completed on schedule

### Time Metrics

**Time Spent:** Actual time spent on completed tasks
**Time Remaining:** Estimated time for remaining tasks
**Total Time:** Time spent + time remaining
**Variance:** Actual vs estimated time

### Quality Metrics

**Acceptance Criteria Met:** Percentage of tasks meeting criteria
**Defect Rate:** Number of defects found
**Rework Rate:** Percentage of tasks requiring rework
**First-Time Pass Rate:** Percentage of tasks passing first verification

### Blocker Metrics

**Active Blockers:** Number of current blockers
**Blocker Severity:** Severity distribution of blockers
**Blocker Resolution Time:** Average time to resolve blockers
**Blocker Impact:** Impact of blockers on timeline

## Tracking Process

### Step 1: Establish Baseline

**Actions:**
- Capture initial plan state
- Record task estimates
- Document dependencies
- Set tracking frequency

**Output:** Baseline for comparison

### Step 2: Collect Progress Data

**Actions:**
- Record task completions
- Track time spent
- Document blockers
- Capture quality metrics

**Frequency:**
- Daily for active projects
- Weekly for standard projects
- Milestone-based for long-running projects

**Output:** Progress data

### Step 3: Compare to Baseline

**Actions:**
- Compare actual vs planned progress
- Identify variances
- Analyze trends
- Detect patterns

**Output:** Variance analysis

### Step 4: Identify Issues

**Actions:**
- Identify blockers
- Identify quality issues
- Identify timeline risks
- Identify resource issues

**Output:** Issue list

### Step 5: Report and Communicate

**Actions:**
- Create progress report
- Communicate to stakeholders
- Highlight risks and blockers
- Recommend actions

**Output:** Progress report

## Progress Reporting

### Daily Progress Report

**Content:**
- Tasks completed today
- Tasks in progress
- Blockers and issues
- Progress against plan
- Next steps

**Format:** Brief, actionable

**Audience:** Team, project manager

### Weekly Progress Report

**Content:**
- Tasks completed this week
- Overall progress percentage
- Timeline variance
- Blockers and risks
- Adjustments made
- Plan for next week

**Format:** Comprehensive, detailed

**Audience:** Team, stakeholders, management

### Milestone Progress Report

**Content:**
- Milestone completion status
- Tasks completed for milestone
- Acceptance criteria verification
- Issues and blockers
- Lessons learned
- Next milestone plan

**Format:** Thorough, milestone-focused

**Audience:** All stakeholders

## Variance Analysis

### Positive Variance

**Definition:** Ahead of schedule or under budget

**Analysis:**
- Identify causes
- Determine if sustainable
- Consider re-estimating future tasks
- Update plan if appropriate

### Negative Variance

**Definition:** Behind schedule or over budget

**Analysis:**
- Identify root causes
- Assess impact on timeline
- Consider mitigation strategies
- Update plan if necessary

### Trend Analysis

**Purpose:** Identify patterns in progress data

**Techniques:**
- Moving average of completion rate
- Trend in time variance
- Blocker frequency trends
- Quality metric trends

**Use:**
- Predict future performance
- Identify systemic issues
- Enable proactive intervention

## Blocker Management

### Blocker Identification

**Sources:**
- Progress tracking
- Team reports
- Stakeholder feedback
- Automated monitoring

### Blocker Classification

**Severity Levels:**
- **Critical:** Blocks all progress, immediate action required
- **High:** Significant impact, urgent action required
- **Medium:** Moderate impact, action required soon
- **Low:** Minor impact, action when convenient

**Types:**
- Technical blockers
- Resource blockers
- Dependency blockers
- Decision blockers
- External blockers

### Blocker Resolution

**Process:**
1. Document blocker
2. Assign severity
3. Assign owner
4. Define resolution approach
5. Set timeline
6. Track progress
7. Verify resolution

## Common Mistakes

### Not Tracking Progress

**Pattern:** No systematic progress tracking

**Problem:** No visibility into execution, issues discovered late

**Fix:** Implement regular progress tracking with defined metrics

### Tracking Too Infrequently

**Pattern:** Tracking progress only at milestones

**Problem:** Issues accumulate, late detection

**Fix:** Track progress regularly (daily for active projects)

### Tracking Wrong Metrics

**Pattern:** Tracking metrics that don't matter

**Problem:** Misleading data, poor decisions

**Fix:** Track meaningful metrics aligned with project goals

### Not Acting on Data

**Pattern:** Collecting data but not using it

**Problem:** Tracking wasted, no improvement

**Fix:** Use data to drive decisions and interventions

# Terraform Module Patterns

## Module anatomy

```
modules/vpc/
├── main.tf        ← resource definitions
├── variables.tf   ← input declarations (no defaults for required inputs)
├── outputs.tf     ← exported values
├── versions.tf    ← required_providers + terraform version constraint
└── README.md      ← auto-generated by terraform-docs
```

## Deep module principle

A deep module hides complexity behind a minimal, stable interface.

**Shallow (avoid):**
```hcl
module "subnet" {
  source             = "./modules/subnet"
  vpc_id             = var.vpc_id
  cidr_block         = var.cidr_block
  availability_zone  = var.availability_zone
  map_public_ip      = var.map_public_ip
  tags               = var.tags
  # 12 more variables...
}
```

**Deep (prefer):**
```hcl
module "vpc" {
  source      = "./modules/vpc"
  name        = "prod"
  cidr        = "10.0.0.0/16"
  az_count    = 3
  # module decides subnet sizing, NAT gateway, route tables internally
}
```

## Variable hygiene

- Required inputs: no `default`.
- Optional inputs: provide sensible `default`.
- Sensitive inputs: mark `sensitive = true` (prevents output in plan).
- Validation blocks on any input with business rules:
```hcl
variable "environment" {
  type = string
  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "environment must be dev, staging, or prod."
  }
}
```

## Output hygiene

- Export only what callers need — no dumping every attribute.
- Mark sensitive outputs `sensitive = true`.
- Document outputs with `description`.

## Versioned module references

```hcl
# Good — pinned tag
module "vpc" {
  source  = "git::https://github.com/org/infra-modules.git//vpc?ref=v2.1.0"
}

# Bad — floating ref
module "vpc" {
  source  = "git::https://github.com/org/infra-modules.git//vpc?ref=main"
}
```

## Common anti-patterns

| Anti-pattern | Why bad | Fix |
|-------------|---------|-----|
| `count` on modules with complex dependencies | Index shifts cause destroy+recreate | Use `for_each` with string keys |
| Hardcoded region in module | Breaks multi-region deploy | Pass as variable |
| `terraform apply -auto-approve` in CI | No human gate for prod destroys | Gate on `plan` output; require approval for destroys |
| Giant root module (everything in one dir) | No reuse, hard to test | Extract child modules per concern |
| `depends_on` on modules | Hides missing output wiring | Wire outputs explicitly |

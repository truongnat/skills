# JavaScript — integration map

| Skill | Combine when |
|-------|----------------|
| **`typescript-pro`** | Types, `strict` config, generics, narrowing — not hand-wavy `any`. |
| **`testing-pro`** | Regression tests for async ordering, timers, mocks, ESM/CJS test runner setup. |
| **`security-pro`** | `eval`, dynamic `Function`, prototype pollution, unsafe deserialization. |
| **`nodejs`-adjacent** | **`nestjs-pro`**, **`nextjs-pro`** for framework-specific server/runtime patterns. |
| **`react-pro`** | Client bundles, hydration, browser-only APIs — not generic JS service code. |
| **`code-packaging-pro`** | `package.json` `exports`, dual ESM/CJS publishing. |

**Boundary:** **`javascript-pro`** owns language/runtime semantics; framework skills own conventions and lifecycle.

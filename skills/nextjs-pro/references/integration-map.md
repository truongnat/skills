# Integration map — nextjs-pro

| Combined skill | Why | `nextjs-pro` owns | Other skill owns |
|----------------|-----|-------------------|------------------|
| **`react-pro`** | Client UI | `"use client"` placement, hydration | Hooks, a11y, component patterns |
| **`security-pro`** | Exposure surface | Middleware, env boundaries, Server Action abuse | Threat model, CSP, auth policy |
| **`testing-pro`** | E2E/RTL | App Router routing in tests, server vs client | CI, pyramid, flakiness |
| **`deployment-pro`** | Prod config | `output`, runtime, image domains | Infra, pipelines |
| **`network-infra-pro`** | CDN, DNS, TLS at edge | `Cache-Control`, origin routing awareness | VPC/LB detail |
| **`caching-pro`** | CDN vs browser vs origin semantics | Fetch cache & invalidation narrative | Stampede policies |
| **`postgresql-pro`** | Server-side data layer | Queries in Server Components / Actions | Schema, migrations |
| **`typescript-pro`** | Typed routes, Next config typings | Strict TS in app/ | Compiler options |
| **`javascript-pro`** | Edge bundle limits, runtime subsets | Pure JS constraints in middleware | Language semantics |
| **`auth-pro`** | Session/JWT/OAuth flows | Middleware + cookie patterns | Identity protocols |

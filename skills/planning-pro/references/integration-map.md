# Planning — integration map

| Skill | When |
|-------|------|
| **`business-analysis-pro`** | Clarify requirements, acceptance criteria, and traceability before locking a schedule. |
| **`strategic-consulting-pro`** | Portfolio-level sequencing and initiative prioritization. |
| **`deployment-pro`** | Release windows, canary, rollback checkpoints, cutover sequencing. |
| **`ci-cd-pro`** | Pipeline milestones, branch strategy impact on integration cadence. |
| **`testing-pro`** | Test strategy, environments, and quality gates aligned to milestones. |
| **`feedback-pro`** | Review findings → backlog priorities and plan adjustments. |
| **`security-pro`** | Compliance / security gates that constrain sequencing (e.g., data handling before exposure). |
| **`bug-discovery-pro`** | Risk of latent defects affecting schedule — buffer and hardening phases. |
| Domain **`*-pro` skills** | Technical depth per phase (backend, frontend, data, infra). |

**Handoff rule:** When the dominant question leaves **executable sequencing** and becomes **domain implementation**, route to the domain skill — **`planning-pro`** stays at milestones, dependencies, and controls.

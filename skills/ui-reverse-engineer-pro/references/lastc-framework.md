# L.A.S.T.C Framework — Fast UI Analysis

Use this framework every time you view a UI image.

## L — Layout

- What is the overall structure?
- Row/column/grid?
- Which block is most important?
- What is the visual hierarchy?

**Questions to answer:**
- How many major sections?
- What is the primary layout direction?
- Which section dominates visual weight?
- How are sections arranged?

---

## A — Alignment

- Left-aligned or center-aligned?
- Are elements on the same axis?
- Is there visual tension from misalignment?

**Questions to answer:**
- Is text left, center, or right aligned?
- Do elements share a common baseline?
- Is there visual grid alignment?
- Are there any alignment breaks?

---

## S — Spacing

- Spacing within the same group?
- Spacing between groups?
- Is the spacing rhythm even?

**Questions to answer:**
- What is the smallest spacing unit?
- What is the largest spacing unit?
- Is there a consistent scale?
- Are groups clearly separated?

---

## T — Typography

- Is hierarchy clear?
- What is primary text?
- What is supporting text?

**Questions to answer:**
- Which text is the title?
- Which text is the body?
- Which text is metadata?
- Is there visual weight differentiation?

---

## C — Components

- What component types exist?
- Which parts can be reused?
- What are the component boundaries?

**Questions to answer:**
- Can this be a Card component?
- Can this be a Button component?
- Can this be a ListItem component?
- What are the reusable patterns?

---

## Quick Application

When looking at any UI image, mentally run through L-A-S-T-C in 30 seconds:

1. **L:** What's the layout? (2 seconds)
2. **A:** How aligned is it? (5 seconds)
3. **S:** What's the spacing rhythm? (10 seconds)
4. **T:** What's the typography hierarchy? (8 seconds)
5. **C:** What are the components? (5 seconds)

Total: 30 seconds for structured analysis.

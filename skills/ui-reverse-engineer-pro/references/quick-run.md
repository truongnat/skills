# Quick Run — 5-Minute Fast Track

When busy, use this condensed version.

## In the first 5 minutes:

### 1. Look at the image and answer:

- How many sections does this screen have?
- What is the primary CTA?
- Where do eyes look first?

### 2. Write down:

- Parent layout
- 3-7 main components
- Temporary spacing scale to use

### 3. Build skeleton

- No style yet
- No icons
- No effects

### 4. Compare visually

- Is the rhythm correct?
- Is it cramped?
- Does it look like "na-na image but stupid"?

### 5. Only then polish

---

## Quick Checklist

**Image analysis (30 seconds):**
- [ ] Sections identified
- [ ] Primary focus found
- [ ] Layout direction noted

**Structure (2 minutes):**
- [ ] Parent layout defined
- [ ] 3-7 components named
- [ ] Spacing scale picked

**Skeleton (2 minutes):**
- [ ] Layout built (no style)
- [ ] Spacing applied
- [ ] Grouping correct

**Review (30 seconds):**
- [ ] Rhythm check
- [ ] Cramp check
- [ ] Clone check

**Polish (remaining time):**
- [ ] Typography applied
- [ ] Components styled
- [ ] Visual details added

---

## When to Use Quick Run

- Tight deadline
- Simple layout
- Familiar patterns
- Reference is clean and clear

## When NOT to Use Quick Run

- Complex layout
- Ambiguous hierarchy
- Unfamiliar patterns
- Reference is messy or unclear
- Production-critical UI

Use the full 7-step process in those cases.

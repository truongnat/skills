# Citations, link rot, and official docs

## Citations

- For **user-facing** answers: **link** to **canonical** URLs (stable paths: `/docs/en/...` where possible).
- **Quote** API names and **version** when behavior differs by release.
- **Distinguish** — Your **summary** vs **verbatim** doc quote (short fair-use excerpts).

## 404 and moved docs

- Official docs **reorganize**; if a **404** — search the vendor’s site with **exact symbol** (function/class name), not only the old path.
- **Archive.org** (sparingly) for **historical** behavior only — not as a substitute for current docs.

## Product self-knowledge / KB

- When a project’s **`knowledge-base`** is **stale** vs `documents/`, **rebuild** the index (`build_kb`) after edits — **not** a “research” shortcut but a **maintenance** requirement (**`repo-tooling-pro`**).

## Limits

- **Paywalled** or **auth** — do not guess; ask the user for access or paste.
- **Legal** — **not** legal research; cite sources for **technical** claims only unless user asks otherwise.

# Web research — decision tree

## Source priority

- **Official docs / spec** > vendor blog > tertiary tutorial.
- **404 or moved** → site search, then archive with caveat.

## Freshness

- **Fast-moving framework** → filter by year + version in query.

## Local vs web

- **Repo has `build-kb`** → query **`query-kb`** before expensive browsing.
- **KB hit looks authoritative** → still verify **versioned** facts on official docs when the stack moves fast — stale embeddings are a first-class risk.

## Evidence discipline

- **Do not fabricate** URLs, quotes, version numbers, or benchmark results.
- Prefer **“per official docs as of [date]”** when the web may change.

---

## Citation hygiene

- Every **non-obvious** API claim should trace to **verifiable** source (fetched page or doc section).
- Distinguish **community opinion** from **normative spec** — **`web-research-verification-system-model.md`**.

---

## Uncertainty language

When sources conflict, state **conflict** and what you verified — **`failure-modes-detection-mitigation.md`**.

---

## Review checklist

- [ ] Primary or triangulated secondary for API facts — **`workflow-sources-and-evaluation.md`**.
- [ ] Version/year scoped for fast-moving topics — **`decision-tree.md`**.
- [ ] KB answers sanity-checked if repo has embeddings — **`integration-map.md`**.
- [ ] No invented links — above citation rules.
